2 · Reasoning — OpenAI
OPENAI · RESPONSES API
🧠 OpenAI one model, five jobs
Text + vision in one model. The same model writes the SQL, reads the scans, reads the sketches and designs the charts — every output validated before it acts.
🗄 Natural Language → SQL job 01
Reads the live schema, writes exactly one Databricks statement, and explains it in a sentence.
🏗 Natural Language → DDL job 02
CREATE SCHEMA · CREATE TABLE · SHOW · DESCRIBE — creation is language too.
👁 Vision — page → table job 03
A scanned invoice is a picture, not data. Vision reads the table straight off the page image.
📊 Two agents design the chart job 04
A VizQL expert reasons over a live data profile to design the chart that fits; a reviewer agent checks the rendered image and self-heals it if it's wrong.

3 · Govern & Deliver
DATABRICKS · TABLEAU
🛡 SQL Safety Guard before execution
Every statement is classified — read / write / destructive — with string literals stripped so a keyword hidden in a quote can't smuggle through. Destructive waits for a click.
🥉 Bronze raw + lineage
Rows exactly as given, plus _ingested_at and _source.
🥈 Silver deduped + typed
Duplicates removed, empty rows dropped, types enforced.
🥇 Gold aggregated
Grouped on a dimension chosen by measuring cardinality — so it actually groups.
📈 Tableau REST .twb → publish → PNG
A real workbook is generated and published, and the rendered chart posts back into Slack.
📧 Email out Resend
Any table or chart can be emailed to any inbox — results reach people who aren't in Slack.
🗃 Unity Catalog Databricks
Catalogs, schemas and tables — governed, and never named after the file.
◂ RETURNED
